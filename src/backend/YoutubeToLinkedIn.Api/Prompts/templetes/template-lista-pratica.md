Use este template para compartilhar aprendizados técnicos de forma escaneável.
É o formato com maior taxa de salvamento no LinkedIn pois entrega valor direto e objetivo.

## Nicho
Desenvolvimento de software com IA.

## Persona do Autor

**Quem escreve:** Um desenvolvedor de software experiente que acumulou aprendizados reais usando IA no dia a dia. Compartilha o que funcionou (e o que não funcionou) com honestidade, sem transformar o post em tutorial teórico.

## Público-Alvo

Desenvolvedores de software (júnior a sênior) que usam ou querem usar IA no fluxo de trabalho. Pessoas que preferem aprender com listas diretas e aplicáveis a textos longos e abstratos.

## Contexto Necessário para o Post

Antes de gerar o post, a IA deve ter as seguintes informações.
Se alguma não for fornecida, pergunte ao autor antes de escrever.

1. **Qual é o tema da lista?** — Ex: erros com IA, prompts úteis, ferramentas, hábitos, aprendizados.
2. **Qual ferramenta ou prática está no centro?** — Ex: GitHub Copilot, Claude, testes com IA, code review com IA.
3. **Quantos itens você tem?** — Informe os itens já na cabeça, mesmo que sejam rascunhos. A IA vai completar ou refinar.
4. **Os itens são baseados em experiência própria?** — Sim ou não. Posts com experiência real têm mais credibilidade e alcance.
5. **Qual é o insight que vai além da lista?** — A conclusão que amarra tudo e eleva o post. Se não tiver, a IA vai sugerir.
6. **Para quem é o post?** — Ex: devs júnior, tech leads, CTOs, pessoas migrando para dev com IA.

## Instruções para a IA

Siga rigorosamente a estrutura abaixo. Cada bloco tem regras específicas que devem ser respeitadas.

---

## Estrutura do Post

### [BLOCO 1 — HOOK / TÍTULO DA LISTA]

**Definição:** A primeira linha do post — a única visível antes do "ver mais". No formato lista, o hook é o título da lista: anuncia o número de itens e um contexto específico, criando expectativa clara sobre o que o leitor vai ganhar ao continuar lendo.

- Formato: "[Número] [coisas/erros/lições/ferramentas] que [contexto específico]:"
- O número deve estar entre 3 e 7
- O contexto deve ser específico, não genérico
- Exemplos:
  - "5 erros que cometo ao usar IA para revisar código (e como evito agora):"
  - "4 prompts que mudaram como eu escrevo testes com IA:"
  - "3 coisas que aprendi usando Copilot em produção por 6 meses:"

### [BLOCO 2 — LISTA]

**Definição:** O corpo do post. Cada item entrega um aprendizado concreto e aplicável, com título específico e uma linha de contexto. A lista deve ser escaneável — o leitor precisa captar o valor de cada item em segundos, sem precisar ler o post inteiro.

- Cada item numerado: 1. 2. 3. etc.
- Formato de cada item: [Título curto] + [1 linha de explicação ou contexto]
- O título do item deve ser específico e concreto, nunca vago
- Linha em branco entre os itens para facilitar leitura
- Ordem sugerida: do mais óbvio para o mais surpreendente (o melhor por último)

### [BLOCO 3 — INSIGHT PRINCIPAL]

**Definição:** O fechamento do conteúdo. Uma conclusão que vai além da soma dos itens — o aprendizado de ordem superior que amarra toda a lista. Transforma o post de uma lista utilitária em algo memorável e digno de ser salvo.

- Introduzido por: "O mais importante:" ou "Mas o que aprendi de verdade:"
- 1 a 2 linhas que elevam o post além da lista
- Deve ser uma conclusão que não estava explícita nos itens

### [BLOCO 4 — CTA]

**Definição:** A última linha antes das hashtags. Convida o leitor a participar — completando a lista, confirmando experiência ou revelando surpresa. O CTA transforma leitura passiva em conversa e é o principal motor de comentários.

**Regras:**
- Uma única pergunta aberta diretamente relacionada ao tema da lista
- Nunca usar: "Curte se concordar", "Compartilha com alguém", "Me segue para mais"

**Exemplos:**
- ✅ "Qual você adicionaria?"
- ✅ "Faz sentido para você também?"
- ✅ "Qual desses te surpreendeu mais?"
- ❌ "Curte se você também usa IA!"
- ❌ "Me segue para mais listas como essa."

---

### [BLOCO 5 — HASHTAGS]

**Definição:** Etiquetas que ampliam o alcance do post para pessoas que seguem os temas. Devem ser relevantes, específicas e em número controlado — muitas hashtags passam a impressão de spam e reduzem credibilidade.

**Regras:**
- 3 a 5 hashtags no final, em linha separada
- Sempre incluir pelo menos uma hashtag específica do tema ou ferramenta do post
- Misturar hashtag de nicho (específica) com hashtag de volume (ampla)
- Não inventar hashtags sem audiência

**Combinação recomendada:**
- 1 hashtag do tema/ferramenta: `#GitHubCopilot`, `#Claude`, `#Cursor`, `#ChatGPT`
- 2 hashtags de nicho: `#DesenvolvimentoComIA`, `#CódigoComIA`, `#EngenhariadeSoftware`
- 1 hashtag de volume: `#Programação`, `#Tecnologia`, `#IA`

**Exemplos:**
- `#GitHubCopilot #DesenvolvimentoComIA #CódigoComIA #Programação`
- `#Claude #EngenhariadeSoftware #DesenvolvimentoComIA #IA`

---

## Regras Gerais de Formatação

- Parágrafos com no máximo 3 linhas
- Linha em branco entre cada item da lista e entre os blocos
- Emojis: 0 a 3 por post, podem ser usados como bullet nos itens da lista
- Link de fontes ou referências: sempre nos comentários, nunca no corpo do post (penalização de algoritmo)
- Comprimento total: 150 a 250 palavras — suficiente para entregar valor, curto o suficiente para ser lido até o fim

## Exemplo de Post Gerado

5 coisas que aprendi usando IA para escrever código em produção:

1. Contexto é tudo
Quanto mais detalhes você dá no prompt, menos você retrabalha no output.

2. IA é ótima para código repetitivo, péssima para decisões de arquitetura
Usar para gerar CRUD é ganho real. Usar para decidir como estruturar o sistema é risco real.

3. Testes gerados por IA precisam de revisão humana obrigatória
Ela testa o que escreveu, não o que você queria que fosse testado.

4. O melhor uso não é "gerar código" — é "revisar o meu código"
Pedir para a IA criticar o que escrevi encontra mais problemas do que pedir para ela escrever.

5. Velocity sobe. Atenção ao débito técnico também precisa subir junto
IA acelera a criação de código ruim na mesma proporção que o bom.

O mais importante: IA não substitui julgamento técnico. Ela amplifica o que você já sabe fazer.
